# Setup Guide for Parents

> Everything you need to get Google Antigravity for Kids running — from installation to handing over the laptop.

---

## Prerequisites

You'll need two things installed on the computer your kid will use:

| Software | What It Is | Where to Get It |
|---|---|---|
| **Google Antigravity IDE** | The AI-powered code editor | [antigravity.google](https://antigravity.google) |
| **Node.js 18+** | JavaScript runtime (needed to run generated apps) | [nodejs.org](https://nodejs.org/) — download the LTS version |

Both are free. Installation takes about 5 minutes total.

### Verify Node.js is installed

Open a terminal (Terminal on Mac, Command Prompt on Windows) and type:

```bash
node --version
```

You should see something like `v20.11.0`. Any version 18 or higher works.

## Installation

### Step 1: Download this project

```bash
git clone https://github.com/dimasyankauskas/GoogleAntigravityForKids.git
```

Or click the green **Code** button on GitHub → **Download ZIP**, then unzip it.

### Step 2: Get a Free Gemini API Key

Kids can use their Google account to get an API key:

1. Go to [Google AI Studio](https://aistudio.google.com/)
2. Sign in with a Google account
3. Click **Get API key** on the left menu
4. Click **Create API key**
5. Copy the long string of letters and numbers

### Step 3: Open in Google Antigravity

1. Launch Google Antigravity
2. Go to **File → Open Folder**
3. Select the `GoogleAntigravityForKids` folder
4. Paste the API key when prompted by Antigravity's setup (or set `GEMINI_API_KEY` in settings)
5. Wait a moment for the AI to initialize

### Step 4: Prepare the workspace

The terminal panel at the bottom shows build output. Your kid doesn't need to see it:

1. Look for the **Terminal** panel at the bottom of the screen
2. Click its header bar to **collapse** it
3. The chat panel on the right is all they need

### Step 4: Hand over the laptop

That's it. When your kid opens the chat and says hello, the AI will:
1. Ask their name
2. Ask what kind of apps they like
3. Ask their visual preference (colorful, minimal, dark, retro, or playful)
4. Save their profile so it remembers next time

From here, they just describe what they want to build.

## How It Works (Technical Overview)

### What's actually happening under the hood

When your kid says "build me a mood tracker," here's what the AI does automatically:

1. **Asks clarifying questions** — confirms the idea, picks a visual style
2. **Creates a new project** — sets up a React + TypeScript project using Vite (a fast build tool)
3. **Generates all the code** — complete, functional, styled components
4. **Installs dependencies** — runs `npm install` silently
5. **Starts the app** — runs the development server in the background
6. **Opens the browser** — navigates to `localhost:5173` where the app is running

Your kid sees none of these steps. They just see the app appear in their browser.

### What technology is used

| Technology | Why | Your Kid Sees It? |
|---|---|---|
| **React** | Popular UI framework, great for interactive apps | No |
| **TypeScript** | Type-safe JavaScript — fewer bugs | No |
| **Vite** | Fast build tool — apps start in seconds | No |
| **Vanilla CSS** | Real styling, no framework dependency | No |
| **localStorage** | Browser storage — fast reads for UI state | No |
| **JSON file storage** | Persistent disk storage — survives everything | No |

All of this is invisible. The AI makes every technical decision.

### Where the code lives

```
GoogleAntigravityForKids/
├── src/                    ← The app's source code (generated by AI)
│   ├── App.tsx             ← Main component
│   ├── components/         ← UI components
│   ├── index.css           ← Styling
│   └── main.tsx            ← Entry point
├── package.json            ← Project dependencies
├── GEMINI.md               ← AI persona configuration
└── .agent/                 ← AI behavior rules
```

After building an app, all the source code is in the project folder. Your kid owns it completely — there's no cloud service, no account, no subscription.

## How the AI Behaves

This project configures the AI with specific rules:

### It speaks in plain language
"Your app is running" instead of "The dev server started on port 5173."

### It makes all technical decisions
It never asks "Should I use React or Vue?" or "Do you want Tailwind?"

### It fixes errors silently
If the code doesn't compile, the AI fixes it automatically. Your kid never sees an error message. If it can't fix something after 2 attempts, it rolls back to the last working version.

### It remembers preferences
The first time your kid uses it, the AI saves their name, interests, and visual style preference to a file called `.builder-profile.json`. Every future session uses these settings.

### It saves data to disk
Any app that involves user input (journal entries, scores, settings) automatically saves to JSON files in the project's `data/` directory. Data survives browser clears, page refreshes, and system reboots. It's a plain file your kid can even open and look at.

## Troubleshooting

### "Nothing happened when I said /build"

Make sure you opened the `GoogleAntigravityForKids` folder as the workspace in Antigravity (not a parent folder).

### The app didn't open in the browser

Try manually opening your browser and going to: `http://localhost:5173`

If that doesn't work, the dev server may not have started. In Antigravity, look for the terminal panel and check for errors.

### The AI is talking like a programmer

The AI personas are configured by the `GEMINI.md` file. Make sure this file exists in the root of the project folder. If it's missing, re-download the project.

### My kid lost their data

Data is stored in JSON files inside the project's `data/` folder. It survives browser clears and restarts. Data can only be lost if:
- The `data/` folder is manually deleted
- The entire project is deleted
- The app is rebuilt from scratch (the AI will warn before doing this)

### The AI generated an ugly app

Try asking: "Pick a different vibe" or name a specific one: "Make it dark" / "Make it colorful." The five available styles are: **colorful**, **minimal**, **dark**, **retro**, and **playful**.

### Can my kid deploy the app to the internet?

Yes, but that requires additional setup (a hosting service like Vercel or Netlify). This is beyond the scope of this guide but is easy to do later. The code is standard React — it works with any hosting provider.

## FAQ

**Is this safe?**
Yes. Everything runs locally. No data is sent to the cloud. No accounts are needed. Your kid can't accidentally break your system — the AI is configured to only work within the project folder.

**Does my kid need to know how to code?**
No. That's the whole point. They describe what they want in plain English and the AI handles everything else.

**Can they learn to code from this?**
Absolutely. When they're curious, they can ask the AI "How does this work?" and it will explain the code in plain language. The code is always visible — it's not a black box.

**Is the AI the same one in regular Antigravity?**
Yes — it's the same Google Gemini AI. This project just changes *how* it talks and *what decisions* it makes automatically. Think of it as a costume change, not a brain swap.

**Can multiple kids use the same installation?**
Currently, the builder profile is shared. If multiple kids use it, they'll share the same preferences. A future update may add multi-profile support.

**What happens if Google changes Antigravity?**
This project is a configuration layer on top of Antigravity, not a fork. If Antigravity updates, this project should continue working. Major breaking changes would require an update to this repo.
